v0.1 将验证最小可玩迷宫闭环：

- 格子迷宫；
- 回合制移动；
- 1 种怪物；
- 普通攻击；
- 1 种回复药；
- 楼梯和下楼；
- HP、楼层、日志；
- 到达第 3 层胜利；
- HP 为 0 失败。

## Web Demo 技术约束

- 优先使用 HTML + CSS + 原生 JavaScript；
- 不引入 React、Phaser、Three.js 或复杂 npm 依赖，除非明确允许；
- 所有核心数值优先放入 `Data/config/`；
- Demo 输出放在 `Builds/web-demo/`；
- 工具脚本放在 `Tools/`；
- 测试脚本放在 `Tests/`。

## 进入 Godot Demo 的条件

- [ ] Web Demo 核心玩法已跑通；
- [ ] 玩家能理解格子回合制；
- [ ] 玩家能理解攻击、拾取、使用道具和下楼；
- [ ] UI 信息层级清楚；
- [ ] 核心反馈足够明确；
- [ ] 数值压力初步合理；
- [ ] 死亡原因可以被玩家理解；
- [ ] 策划确认玩法值得迁移到 Godot。
