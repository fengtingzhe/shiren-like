## Demo 阶段系统优先级

```text
回合制闭环 > 地图与移动 > 怪物与战斗 > 道具解局 > UI 可读性 > 随机与长期内容
```

当前最优先验证：

```text
格子移动 + 回合推进 + 怪物追击 + 普通攻击 + 回复药 + 楼梯下楼
```

## v0.1 系统范围

v0.1 只做最小可玩迷宫版：

- 回合系统；
- 格子地图；
- 玩家移动；
- 1 种怪物；
- 普通攻击；
- 1 种回复药；
- 楼梯；
- HP；
- 当前层数；
- 行动日志；
- 简单胜负。

v0.1 明确不做：

- 饥饿；
- 完整背包；
- 未鉴定；
- 装备；
- 陷阱；
- 怪物屋；
- 壶；
- 商店；
- 据点；
- Godot 工程。

## 核心系统边界

### 回合系统

- 每次玩家有效行动后推进一回合；
- 怪物在同一回合内按顺序行动；
- UI 操作不应误推进回合；
- 关键行动进入日志。

### 地图系统

- v0.1 可使用固定或半随机地图；
- 地图必须有墙、地板、玩家起点、怪物、回复药、楼梯；
- 地图不应生成无解局；
- 后续再加入真正随机房间和通道。

### 怪物系统

- 怪物不是简单数值堆叠；
- v0.1 先做基础近战怪；
- v0.4 后逐步加入远程、偷窃、催眠、爆炸、隔格攻击等规则怪。

### 道具系统

- 道具必须服务解局；
- v0.1 只做回复药；
- v0.2 加入卷轴、杖、食物和基础背包；
- 后续加入未鉴定与壶。

### 死亡复盘

- 死亡必须说明主要原因；
- v0.1 至少显示“被某怪物击败 / HP 归零”；
- v0.5 后加入最近回合记录。
