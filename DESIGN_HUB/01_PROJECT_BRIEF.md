# 01_PROJECT_BRIEF：项目定位

## 游戏一句话描述

一款现代化 Mystery Dungeon Roguelike Demo：玩家在随机迷宫中逐层探索，以回合制格子移动、道具组合、怪物机制、陷阱、饥饿压力和死亡复盘为核心，体验“每一步都是决策”的高密度迷宫冒险。

---

## 游戏类型

- 主类型：Mystery Dungeon / Traditional Roguelike / 回合制格子迷宫
- 副类型：道具策略 / 随机迷宫 / 轻硬核冒险 / 单局挑战
- 参考方向：
  - `风来的西林`：参考其格子回合制、随机迷宫、未鉴定、道具解局、死亡代价和故事性失败。
  - `不可思议迷宫`类作品：参考逐层探索、怪物规则、道具组合和资源压力。
  - 现代 Roguelike 可读性优化：参考危险提示、死因复盘、图鉴记录、操作便利性和新手学习曲线。

不做方向：

- 不做动作 Roguelike；
- 不做刷数值碾压；
- 不做抽卡角色养成；
- 不做大地图开放世界；
- 不做复杂基地建设；
- 不做商业化优先的强付费系统。

---

## 目标平台

- Web Demo：当前优先，用于快速验证核心玩法。
- Godot Demo：Web Demo 验证后迁移。
- PC：适合展示和调试。
- Android / iOS：后续可评估。

---

## 目标用户

- 喜欢 Roguelike、迷宫探索、策略道具和死亡复盘的玩家；
- 能接受一定失败代价，但不希望死得不明不白的泛用户；
- 对复杂动作操作不敏感，更重视局势判断和策略解法的玩家。

---

## 产品设计理念

本项目的核心不是复刻某一款作品，而是提炼 Mystery Dungeon 的核心乐趣：

```text
每一步都是决策；
每个道具都有多种用途；
每种怪物都改变局势；
随机生成带来新鲜感；
死亡产生故事和学习；
失败要有代价，但必须可复盘。
```

---

## 当前 Demo 目标

当前阶段先完成 Web Demo，验证：

- 玩家是否能理解格子回合制：玩家行动一次，怪物行动一次；
- 玩家是否能理解迷宫目标：探索、找楼梯、进入下一层；
- 玩家是否能通过攻击和道具解决最基础危机；
- UI 是否能清楚表达 HP、楼层、道具和行动日志；
- 死亡或胜利是否有清楚反馈；
- 该方向是否值得进入 Godot Demo 阶段。

---

## 长期版本目标

长期目标是形成一款轻量但有深度的 Mystery Dungeon Roguelike：

- 多主题迷宫；
- 多种怪物规则；
- 卷轴、杖、药水、食物、装备、壶等道具系统；
- 未鉴定与诅咒；
- 陷阱、怪物屋、商店；
- 死亡复盘和图鉴沉淀；
- 可重复挑战的单局结构。
