# 05_ECONOMY_AND_BALANCE：数值、资源与平衡方向

本文件用于记录 shiren-like 的资源压力、数值方向、掉落节奏和成长边界。

---

## 核心资源

| 资源 | 来源 | 消耗 / 风险 | 设计目的 | Demo 阶段 |
|---|---|---|---|---|
| HP | 初始值、回复药、后续自然回血 | 怪物攻击、陷阱、饥饿 | 生存核心压力 | v0.1 |
| 楼层深度 | 下楼推进 | 怪物强度、资源压力提高 | 单局进度和成就感 | v0.1 |
| 道具数量 | 地图拾取、宝箱、商店后置 | 使用、丢弃、背包限制 | 解局资源 | v0.1 起 |
| 背包格子 | 初始容量 | 道具占用 | 取舍压力 | v0.2 起 |
| 饥饿 / 满腹度 | 食物 | 回合流逝、特殊效果 | 防止无限刷层，制造探索节奏 | v0.3 起 |
| 回合数 | 玩家行动推进 | 饥饿消耗、怪物刷新后置 | 控制节奏和复盘 | v0.1 起 |
| 经验 / 等级 | 击败怪物 | 死亡清算 | 轻量局内成长 | 后置 |

## v0.1 初版数值草案

| 项目 | 建议值 |
|---|---:|
| 玩家初始 HP | 20 |
| 普通怪 HP | 6 |
| 玩家普通攻击 | 4 |
| 普通怪攻击 | 3 |
| 回复药恢复 | 8 |
| 初始楼层数 | 3 |
| 每层怪物数量 | 2～4 |
| 每层回复药数量 | 1～2 |
| 玩家失败条件 | HP 为 0 |
| 玩家胜利条件 | 到达第 3 层终点 |

这些数值只用于验证，不代表最终平衡。

## v0.2 道具策略数值方向

v0.2 加入基础道具后，重点验证“道具能否改变局势”。

建议道具：

| 道具 | 作用 | 平衡目的 |
|---|---|---|
| 回复药 | 恢复 HP | 基础容错 |
| 食物 | 恢复满腹度 | 为 v0.3 饥饿做准备 |
| 传送卷轴 | 随机传送玩家 | 逃离包围 |
| 睡眠卷轴 | 让房间内怪物睡眠 | 房间控场 |
| 火球杖 | 远程伤害 | 处理远处威胁 |
| 换位杖 | 与目标交换位置 | 走位解局 |
| 识别卷轴 | 鉴定道具 | 为未鉴定做准备 |

## v0.3 饥饿数值方向

饥饿不是单纯惩罚，而是探索节奏压力。

建议：

- 满腹度初始 100；
- 每 10 回合下降 1；
- 满腹度低于 25 时自然回血降低或停止；
- 满腹度为 0 时每若干回合扣 HP；
- 食物恢复 40～60 满腹度；
- 玩家应在“继续搜刮”和“尽快下楼”之间产生取舍。

---

## 掉落与楼层压力原则

- 前 1～2 层应保证玩家学会基本操作，不应过早击杀玩家；
- 中段开始让玩家感到资源紧张；
- 道具掉落不能太多，否则无脑；
- 道具掉落不能太少，否则失去解局乐趣；
- 怪物强度应更多来自机制和组合，而不是纯数值堆高；
- 死亡原因应主要来自决策、位置和资源管理，而不是不可预判的随机坑杀。

---

## 不做的经济方向

当前项目不做：

- 金币单资源建造经济；
- 木材、石头、铁矿等多资源采集；
- 建筑升级经济；
- 抽卡养成经济；
- 强付费复活经济；
- 商业化数值压迫。

## 待验证问题

- 玩家 HP 和怪物攻击是否让前 3 层足够可学？
- 回复药数量是否足够支撑第一次通关？
- 怪物密度是否让迷宫有压力但不拥挤？
- 背包容量应该从多少开始？
- 饥饿应在 v0.3 加入，还是更早加入？
- 随机地图是否会产生无解局？
- 死亡复盘能否让玩家愿意再来一局？
