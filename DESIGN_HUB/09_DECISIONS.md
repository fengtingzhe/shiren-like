### 2026-05-13：当前阶段先做 Web Demo

#### 决策内容

当前阶段继续采用“先 Web Demo，后 Godot Demo”的路线。

v0.x 用 Web Demo 快速验证核心玩法；v1.x 再迁移到 Godot Demo。

#### 决策原因

Web Demo 更适合快速验证格子回合制、怪物追击、道具使用、楼梯推进和基础 UI。等核心体验成立后，再进入 Godot 可以降低返工成本。

#### 状态

已确认。

#### AI 执行要求

AI 不得在 v0.1 阶段直接启动完整 Godot 工程，也不得绕过 Web Demo 直接做复杂垂直切片。

### 2026-05-13：Demo 目标是验证 Mystery Dungeon 最小闭环

#### 决策内容

v0.1 只验证最小可玩迷宫闭环：

```text
格子迷宫 → 回合制移动 → 怪物追击 → 普通攻击 → 回复药 → 楼梯下楼 → 简单胜负
```

#### 决策原因

Shiren-like 玩法复杂度高，必须先确认最基础的移动、回合、战斗和楼层推进成立，再逐步加入道具、饥饿、陷阱、未鉴定、怪物屋等系统。

#### 状态

已确认。

#### AI 执行要求

v0.1 不得一次性加入饥饿、完整背包、未鉴定、装备、陷阱、怪物屋、壶、商店、据点或商业化系统。
