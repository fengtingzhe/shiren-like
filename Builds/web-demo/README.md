# shiren-like Web Demo

当前版本：`v0.7.1 / 装备池补全与平衡版`

这一轮是 `v0.7` 的补丁版，不新增大系统，只补齐高级装备并把装备成长节奏调顺。现在玩家仍然通过站到装备上按 `C` 来装备或替换，但装备池从“基础可用”扩成了“有明显成长终点”。

## 当前系统

- 随机房间与走廊
- 玩家中心镜头
- `Console / Camera / View` 调参
- 饥饿与饱腹度
- 小地图、敌人面板、底部快捷栏
- 四种怪物
  - `Slime`
  - `Goblin Archer`
  - `Sleep Mushroom`
  - `Skeleton Spearman`
- 三种陷阱
  - `Spike Trap`
  - `Warp Trap`
  - `Sleep Trap`
- 一种危险地形
  - `Poison Pool`
- 六种消耗道具
  - 恢复药
  - 食物
  - 传送卷轴
  - 睡眠卷轴
  - 火球杖
  - 换位杖

- `1F`
  - 主要是 `短剑 / 木盾`
  - `长枪 / 铁盾` 只会低概率出现
- `2F`
  - `长枪 / 铁盾` 开始成为主流
  - `火焰剑 / 守护盾` 会极低概率提前出现
- `3F`
  - `火焰剑 / 守护盾` 开始正式进入掉落池
  - 玩家能在终层前获得明显的攻防成长

## 装备规则

- 装备位仍然只有：
  - `weapon`
  - `shield`
- 消耗品仍然是走上去自动拾取
- 装备不会自动拾取
- 站在装备上时，按 `C` 才会装备或替换
- 装备或替换会推进 `1` 回合

## 装备替换与掉落

- 当前槽位为空时，新装备直接进入对应槽位
- 当前槽位已有装备时，旧装备会尝试掉回玩家脚下
- 如果脚下不能放，旧装备会尝试掉到相邻安全格
- 如果附近没有可放置的位置，旧装备会直接消失，并写日志说明
- 不允许装备与怪物、道具、陷阱、危险地形、楼梯重叠

## 数值规则

- 玩家攻击力 = 基础攻击 + 武器加成
- 玩家防御力 = 基础防御 + 盾牌加成
- 怪物攻击会先扣掉防御再结算
- 最低伤害仍然是 `1`
- 陷阱、`Poison Pool` 和饥饿伤害不吃盾牌减免
- 火球杖仍然只使用道具自身伤害，不吃武器攻击加成

## HUD 与日志

- HUD 继续显示：
  - `ATK`
  - `DEF`
  - `武器`
  - `盾牌`
- 高级装备会直接显示为：
  - `火焰剑 +4`
  - `守护盾 +3`
- 日志会明确区分：
  - 装备后数值变化
  - 旧装备掉在脚下
  - 旧装备掉在旁边
  - 旧装备无法放置而消失

## 特别说明

  - 不做火焰特效

  - 不做特殊攻击

  - 不做特殊抗性
  - 不做反伤
  - 不做免疫陷阱
  - 不做耐久

## 运行

在项目根目录运行：

```text
node Tools/web-demo-server.mjs
```

然后打开：

```text
http://127.0.0.1:4173/Builds/web-demo/
```

也可以直接运行：

```text
Tools/run-web-demo-vite.bat
```

默认会打开：

```text
http://127.0.0.1:5173/Builds/web-demo/
```
