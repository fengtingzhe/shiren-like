# shiren-like Web Demo

当前版本：`v0.4.1 / 传统轻斜俯视镜头修正版`

本版本不新增核心玩法系统，重点修正 v0.4 等距 / 菱形镜头带来的操作不便利问题。

本轮选择：

```text
B 方案：75° 传统轻斜俯视视角
```

目标是让画面保留轻微纵深和棋子感，但操作方向重新回到传统 Roguelike / 地牢 RPG 的直觉模式：

```text
按上就是屏幕上移，按下就是屏幕下移，按左就是屏幕左移，按右就是屏幕右移。
```

v0.3 的随机迷宫、饥饿压力、道具、怪物追击、楼梯和胜负条件，以及 v0.4 的 HUD、小地图、敌人面板和快捷栏都继续保留。

---

## 运行方式

在项目根目录运行：

```text
node Tools/web-demo-server.mjs
```

然后打开：

```text
http://127.0.0.1:4173/Builds/web-demo/
```

也可以双击：

```text
Tools/run-web-demo.bat
```

## v0.4.1 镜头修正

- 废弃 v0.4 的主要等距菱形棋盘观感；
- 改为传统轻斜俯视矩形格；
- 画布仍是 16:9 横屏战术地牢布局；
- 地图从上往下看，但 y 轴有轻微压缩，产生纵深感；
- 玩家、怪物、道具保持立起来的棋子感；
- 地图方向重新与屏幕上下左右一致。

核心坐标思路：

```text
cameraMode = "traditional-tilt"
yScale = 0.72
screenX = originX + x * tileW + perspectiveOffset * y
screenY = originY + y * tileH * yScale
```

---

## 操作方向

v0.4.1 的重点是修正操作方向：

- `W` / 上方向键：角色在屏幕中向上移动；
- `S` / 下方向键：角色在屏幕中向下移动；
- `A` / 左方向键：角色在屏幕中向左移动；
- `D` / 右方向键：角色在屏幕中向右移动。

本轮不再要求玩家在脑中把上下左右映射到斜向菱形格。

## 地砖与范围提示

- 地砖从等距菱形改为轻斜俯视矩形格；
- 每个格子边界仍然清楚；
- 蓝色行动范围准确覆盖可行动矩形格；

## UI 布局

v0.4 的 UI 结构保留：

- 左上：玩家头像占位、HP、满腹度、楼层、回合、回复药数量；
- 右上：小地图，显示墙、地板、玩家、楼梯、怪物和道具点；
- 右侧：敌人面板 / 行动序列；
- 底部：快捷栏，显示 H/F/T/Z/R/X 道具、数量、等待和下楼命令；
- Console 入口仍保留通用调试功能。

---

## 保留的玩法

- 每层随机生成房间和走廊；
- 玩家、楼梯、怪物和道具随机分布在合法地板格；
- 每次下楼都会重新生成当前层地图、怪物和道具；
- 满腹度随有效回合下降；
- 满腹度低时提示；
- 满腹度为 0 后造成轻微 HP 压力；
- 食物恢复满腹度；
- 饥饿死亡与怪物死亡原因区分；
- 第 3 层胜利，HP 为 0 失败。

---

## 保留的道具

| 道具 | 快捷键 | 作用 |
|---|---|---|
| 回复药 | H | 恢复 HP，成功使用后推进回合 |
| 食物 | F | 恢复满腹度 |
| 传送卷轴 | T | 随机传送到当前楼层安全地板 |
| 睡眠卷轴 | Z | 让 4 格内怪物睡眠 3 回合 |
| 火球杖 | R | 朝当前朝向发射火球，命中第一个怪物，未命中不消耗 |
| 换位杖 | X | 与当前朝向第一个怪物交换位置，未命中不消耗 |
