# shiren-like Web Demo

当前版本：`v0.4 / 镜头与画面表现调整版`

本版本不新增核心玩法系统，重点把 v0.3 的普通俯视格子原型调整为：

```text
2.5D 斜俯视战术迷宫镜头 + 清晰战斗棋盘 + 现代 Roguelike UI 布局
```

v0.3 的随机迷宫、饥饿压力、道具、怪物追击、楼梯和胜负条件仍保留。

---

## 运行方式

在项目根目录运行：

```text
node Tools/web-demo-server.mjs
```

然后打开：

```text
http://127.0.0.1:4173/Builds/web-demo/
```

也可以双击：

```text
Tools/run-web-demo.bat
```

---

## v0.4 画面调整

- 主画布改为 16:9 横屏战术地牢观感；
- Canvas 2D 使用 `tileToScreen` 等距感坐标转换绘制地牢；
- 地板改为菱形格，墙体改为带高度的暗色块；
- 玩家、怪物、道具和楼梯按格子深度排序绘制，减少遮挡错位；
- 玩家、怪物、道具、楼梯的尺寸和轮廓比 v0.3 更清楚；
- 画面四周保留暗角和边缘氛围，但不牺牲格子可读性。

---

## 战术范围提示

- 玩家附近显示蓝色半透明行动范围；
- 怪物相邻攻击范围显示红色半透明威胁范围；
- 范围叠层先于角色和道具绘制，不遮挡棋子识别；
- 本轮只显示当前格子威胁，不做复杂下一回合预测。

---

## UI 布局

v0.4 将 HUD 调整为更接近现代战术 Roguelike 的布局：

- 左上：玩家头像占位、HP、满腹度、楼层、回合、回复药数量；
- 右上：小地图，显示墙、地板、玩家、楼梯、怪物和道具点；
- 右侧：敌人面板 / 行动序列，显示玩家与最近敌人、敌人 HP 和睡眠状态；
- 底部：快捷栏，显示 H/F/T/Z/R/X 道具、数量、等待和下楼命令；
- Console 入口仍保留通用调试功能。

---

## 保留的 v0.3 玩法

- 每层随机生成房间和走廊；
- 玩家、楼梯、怪物和道具随机分布在合法地板格；
- 每次下楼都会重新生成当前层地图、怪物和道具；
- 满腹度随有效回合下降；
- 满腹度低时提示；
- 满腹度为 0 后造成轻微 HP 压力；
- 食物恢复满腹度；
- 饥饿死亡与怪物死亡原因区分；
- 第 3 层胜利，HP 为 0 失败。

---

## 保留的道具

| 道具 | 快捷键 | 作用 |
|---|---|---|
| 回复药 | H | 恢复 HP，成功使用后推进回合 |
| 食物 | F | 恢复满腹度 |
| 传送卷轴 | T | 随机传送到当前楼层安全地板 |
| 睡眠卷轴 | Z | 让 4 格内怪物睡眠 3 回合 |
| 火球杖 | R | 朝当前朝向发射火球，命中第一个怪物，未命中不消耗 |
| 换位杖 | X | 与当前朝向第一个怪物交换位置，未命中不消耗 |

移动仍使用 WASD / 方向键。`S` 保留为向下移动，因此睡眠卷轴使用 `Z`。
